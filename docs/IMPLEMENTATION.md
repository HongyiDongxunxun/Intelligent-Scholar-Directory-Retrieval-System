# 重构实现说明

## 保留的可验证能力

1. 多字段、布尔条件、年份筛选和稳定游标式结果结构；
2. 从候选文献聚合学者，返回匹配文献、被引量、领域分和证据；
3. 按 `0.70 Rel + 0.20 Str + 0.10 Inf` 装配文献、学者、机构和关键词；
4. 专题结果记录分量、综合分、证据论文、算法版本和缓存状态；
5. 自然语言先解析为结构化草案，经过确认后才执行确定性检索；
6. 实体详情返回属性和关系证据，支持前端侧边栏查看；
7. 保留原实验文档所列七个旧接口的兼容入口，便于已有脚本迁移。

## 有意替换的部分

| 原系统依赖 | 公开重构实现 |
|---|---|
| 远程或本地 MySQL | 固定、内存生成的合成目录 |
| Elasticsearch | 对 60 条合成文献进行确定性内存检索 |
| Redis | 带 30 分钟有效期的进程内专题缓存 |
| 外部大模型 | 可重复的规则解析器，保留解析与确认边界 |
| 真实学者和机构 | `A/U/D` 前缀的合成实体 |

这些替换用于公开审稿验证，不能与历史环境做吞吐或召回量的直接数值比较。

## 评分定义

- `Rel`：实体关联候选文献的相关度聚合并归一化；
- `Str`：实体在候选子图中的连接覆盖度并归一化；
- `Inf`：文献被引量或实体关联文献影响力并归一化；
- `Score = 0.70 * Rel + 0.20 * Str + 0.10 * Inf`。

分量和综合分保留四位小数。相同查询返回稳定顺序，每项携带证据文献 ID。

## 安全边界

- 配置文件不包含数据库、SSH、模型或账户凭证；
- 服务启动时不连接公网或任何数据库；
- AI 解析器不能调用工具或构造任意 URL；
- 输入仅映射到白名单字段、匹配方式和排序类型；
- 公开代码不得用真实数据替换 `FixtureCatalog` 后再提交。
